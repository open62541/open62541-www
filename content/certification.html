Certified Implementation

The sample server built using open62541 has been certified by the OPC Foundation for compliance with the Full Server 2017 profile. This certification demonstrates that open62541 provides a robust, standards-compliant OPC UA implementation suitable for production use.

Certification Details

The OPC Foundation Certified for Compliance logo is a trademark of the OPC Foundation and may be used only by written permission. This certification indicates that our implementation has been tested by an independent certification lab and meets all requirements of the following specification:

Profile:

Full Server 2017

Security Policies:

  • Basic128Rsa15
  • Basic256
  • Basic256Sha256
  • Aes128-Sha256-RsaOaep
  • Aes256-Sha256-RsaPss

Key Features:

  • Client/Server communication
  • Subscriptions and monitored items
  • Method calls
  • Node management
  • Historical data access
  • Events and alarms
  • Security (encryption and user authentication)

About the Certification

The certified implementation is based on the open62541 SDK and demonstrates the full capabilities of the library. Organizations using open62541 can reference this certification when developing their own OPC UA applications.

Important Notes:

  • Certification applies to the reference server implementation built with open62541
  • Applications built with open62541 can leverage the certified codebase
  • Organizations requiring product-specific certification should contact the OPC Foundation Test Lab
  • The certified server implementation is available in the open62541 repository

Additional Features

Beyond the certified features, open62541 also implements:

  • PubSub (UADP/MQTT): Publisher/Subscriber communication pattern
  • JSON encoding: Alternative data encoding format
  • TSN support: Time-Sensitive Networking capabilities

These features are actively maintained and tested but may not yet have official OPC Foundation certification test tools available.
